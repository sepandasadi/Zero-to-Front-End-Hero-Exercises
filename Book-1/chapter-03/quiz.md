# Chapter 3 Quiz: Command Line Basics

Test your command line knowledge!

---

## Question 1
**What command shows your current directory location?**

a) `ls` or `dir`
b) `pwd` or `cd` (without arguments)
c) `where`
d) `location`

<details>
<summary>Answer</summary>
**b) `pwd` or `cd` (without arguments)**

`pwd` (print working directory) on Mac/Linux, or `cd` without arguments on Windows shows your current location.
</details>

---

## Question 2
**What does `cd ..` do?**

a) Goes to the home directory
b) Goes up one level to the parent directory
c) Lists the current directory
d) Goes to the root directory

<details>
<summary>Answer</summary>
**b) Goes up one level to the parent directory**

`..` represents the parent directory, so `cd ..` moves you up one level in the folder hierarchy.
</details>

---

## Question 3
**What's the difference between absolute and relative paths?**

a) There is no difference
b) Absolute paths start from the root directory; relative paths start from current location
c) Absolute paths are faster
d) Relative paths can only go down, not up

<details>
<summary>Answer</summary>
**b) Absolute paths start from the root directory; relative paths start from current location**

Absolute paths give the full path from root (e.g., `/Users/name/Documents`), while relative paths are based on where you currently are (e.g., `../Documents`).
</details>

---

## Question 4
**Which command creates a new directory?**

a) `newdir` or `createdir`
b) `mkdir`
c) `makedir`
d) `md`

<details>
<summary>Answer</summary>
**b) `mkdir`**

`mkdir` (make directory) creates new folders. `md` also works on Windows as a shorthand.
</details>

---

## Question 5
**What does the `~` symbol represent in Unix-based systems?**

a) The current directory
b) The parent directory
c) The home directory
d) The root directory

<details>
<summary>Answer</summary>
**c) The home directory**

`~` is a shortcut for your home directory. `cd ~` takes you home from anywhere.
</details>

---

## Question 6
**Which command lets you create an empty file on Mac/Linux?**

a) `create`
b) `newfile`
c) `touch`
d) `makefile`

<details>
<summary>Answer</summary>
**c) `touch`**

`touch filename` creates an empty file. On Windows, use `type nul > filename`.
</details>

---

## Question 7
**What does `rm -r` do on Unix systems?**

a) Removes a file
b) Renames a folder
c) Recursively removes a directory and its contents
d) Restores deleted files

<details>
<summary>Answer</summary>
**c) Recursively removes a directory and its contents**

`rm -r` (remove recursive) deletes a folder and everything inside it. **Use with caution!**
</details>

---

## Question 8
**Which keyboard shortcut stops a running command?**

a) Ctrl + S
b) Ctrl + Z
c) Ctrl + C
d) Ctrl + X

<details>
<summary>Answer</summary>
**c) Ctrl + C**

Ctrl + C sends an interrupt signal to stop the currently running process.
</details>

---

## Question 9
**What does Tab completion do?**

a) Closes the terminal
b) Autocompletes file and folder names
c) Creates a new tab
d) Indents your code

<details>
<summary>Answer</summary>
**b) Autocompletes file and folder names**

Pressing Tab autocompletes commands, file names, and folder names, saving time and preventing typos.
</details>
