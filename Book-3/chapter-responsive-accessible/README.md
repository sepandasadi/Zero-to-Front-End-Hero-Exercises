## üí° Pro Tips

**Responsive Design Best Practices:**
- Always start mobile-first
- Use relative units (rem, %, vw) over fixed pixels
- Test on real devices, not just DevTools
- Optimize images for different screen sizes
- Keep touch targets ‚â• 44px √ó 44px
- Avoid horizontal scrolling
- Use `clamp()` for fluid typography
- Think about performance on mobile networks

**Accessibility Best Practices:**
- Use semantic HTML first, ARIA only when needed
- Ensure all interactive elements are keyboard accessible
- Provide visible focus indicators
- Test with keyboard only (unplug your mouse!)
- Use screen readers to verify announcements
- Ensure color contrast meets WCAG AA (4.5:1)
- Provide text alternatives for images
- Label all form inputs
- Create logical heading hierarchy
- Test with browser zoom (200%, 400%)

**Common Pitfalls to Avoid:**
- ‚ùå Removing focus outlines without providing alternatives
- ‚ùå Using divs with onClick instead of buttons
- ‚ùå Missing alt text on images
- ‚ùå Low color contrast (test with tools!)
- ‚ùå Tiny text or touch targets on mobile
- ‚ùå Relying on color alone to convey information
- ‚ùå Fixed-width layouts that don't adapt
- ‚ùå Missing viewport meta tag
- ‚ùå Using ARIA incorrectly or unnecessarily
- ‚ùå Not testing with actual screen readers
