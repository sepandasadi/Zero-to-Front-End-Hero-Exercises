# Project 2: Product Landing Page - Evaluation Rubric

**Total Points: 100**

---

## HTML Structure & Semantics (20 points)

### Excellent (18-20 points)
- Uses semantic HTML5 elements appropriately (`<header>`, `<nav>`, `<section>`, `<article>`, `<footer>`)
- Proper heading hierarchy (h1 → h2 → h3) with no skipped levels
- All forms have associated labels
- Alt text is descriptive and meaningful
- Valid HTML (passes W3C validator)

### Good (14-17 points)
- Mostly semantic HTML with minor issues
- Heading hierarchy mostly correct
- Most images have alt text
- 1-2 HTML validation errors

### Needs Improvement (10-13 points)
- Limited use of semantic elements (excessive divs)
- Heading hierarchy has gaps
- Missing alt text on some images
- 3-5 HTML validation errors

### Unsatisfactory (0-9 points)
- Non-semantic HTML throughout
- No consideration of heading hierarchy
- Most images lack alt text
- Multiple validation errors

---

## CSS Layouts & Design (30 points)

### Flexbox (10 points)
- **Excellent (9-10):** Proper use of Flexbox for one-dimensional layouts (navigation, feature cards, buttons)
- **Good (7-8):** Mostly correct Flexbox usage with minor issues
- **Needs Improvement (5-6):** Limited Flexbox usage or incorrect implementation
- **Unsatisfactory (0-4):** No Flexbox or severely broken layouts

### CSS Grid (10 points)
- **Excellent (9-10):** Proper use of Grid for two-dimensional layouts (pricing table, testimonials)
- **Good (7-8):** Mostly correct Grid usage
- **Needs Improvement (5-6):** Limited Grid usage or workarounds
- **Unsatisfactory (0-4):** No Grid or broken layouts

### Visual Design (10 points)
- **Excellent (9-10):** Professional color scheme, consistent spacing, good typography, visual hierarchy
- **Good (7-8):** Decent design with minor inconsistencies
- **Needs Improvement (5-6):** Inconsistent design, poor color choices, spacing issues
- **Unsatisfactory (0-4):** No design consideration, looks unfinished

---

## Responsive Design (20 points)

### Mobile (320-767px) (7 points)
- **Excellent (6-7):** Perfect mobile layout, touch-friendly, readable
- **Good (5):** Minor mobile issues
- **Needs Improvement (3-4):** Significant mobile problems
- **Unsatisfactory (0-2):** Broken on mobile

### Tablet (768-1023px) (6 points)
- **Excellent (5-6):** Smooth transition, good use of space
- **Good (4):** Minor tablet issues
- **Needs Improvement (2-3):** Layout problems
- **Unsatisfactory (0-1):** Broken on tablet

### Desktop (1024px+) (7 points)
- **Excellent (6-7):** Full-featured desktop experience
- **Good (5):** Minor desktop issues
- **Needs Improvement (3-4):** Poor use of space
- **Unsatisfactory (0-2):** Broken on desktop

---

## Functionality & Completeness (15 points)

### Required Sections (10 points)
- **All 7 sections present and functional** (10 points)
- **6 sections present** (8 points)
- **5 sections present** (6 points)
- **4 or fewer sections** (0-4 points)

### Interactive Elements (5 points)
- **Excellent (5):** All buttons, links, and forms work correctly
- **Good (4):** Minor issues with interactivity
- **Needs Improvement (2-3):** Several broken elements
- **Unsatisfactory (0-1):** Most elements non-functional

---

## Accessibility (10 points)

### Color Contrast (3 points)
- **Excellent (3):** All text meets WCAG AA (4.5:1)
- **Good (2):** Minor contrast issues
- **Needs Improvement (1):** Multiple contrast failures
- **Unsatisfactory (0):** No consideration of contrast

### Keyboard Navigation (3 points)
- **Excellent (3):** Fully keyboard navigable with visible focus states
- **Good (2):** Mostly navigable, some focus issues
- **Needs Improvement (1):** Limited keyboard support
- **Unsatisfactory (0):** Cannot navigate with keyboard

### ARIA & Labels (4 points)
- **Excellent (4):** Proper ARIA labels, all inputs labeled, icon buttons accessible
- **Good (3):** Mostly accessible with minor gaps
- **Needs Improvement (2):** Missing several accessibility features
- **Unsatisfactory (0-1):** No accessibility consideration

---

## Code Quality (5 points)

### Organization (2 points)
- **Excellent (2):** Well-organized CSS, clear comments, logical structure
- **Good (1):** Mostly organized
- **Unsatisfactory (0):** Disorganized, no comments

### Efficiency (2 points)
- **Excellent (2):** No repeated code, efficient selectors, good use of CSS
- **Good (1):** Some repetition
- **Unsatisfactory (0):** Highly repetitive, inefficient

### Validation (1 point)
- **Excellent (1):** Valid HTML and CSS (W3C validators)
- **Unsatisfactory (0):** Validation errors

---

## Bonus Points (Up to +10)

- **Extension Features** (+2-5): Implemented stretch goals (animation, video bg, pricing toggle, etc.)
- **Exceptional Design** (+2-3): Professional-quality visual design
- **Performance** (+1-2): Optimized images, fast loading
- **Extra Accessibility** (+1-2): Skip links, reduced motion preferences, etc.

---

## Grading Scale

- **90-100 points:** Excellent - Professional quality, ready for portfolio
- **80-89 points:** Good - Solid work with minor improvements needed
- **70-79 points:** Satisfactory - Meets requirements but lacks polish
- **60-69 points:** Needs Improvement - Missing features or significant issues
- **Below 60:** Unsatisfactory - Does not meet minimum requirements

---

## Self-Assessment Checklist

Before considering your project complete, check these:

### Must-Have
- [ ] All 7 sections implemented
- [ ] Responsive on mobile, tablet, and desktop
- [ ] Valid HTML and CSS
- [ ] All images have alt text
- [ ] Forms have labels
- [ ] Color contrast meets WCAG AA
- [ ] Focus states visible
- [ ] Works in all major browsers

### Should-Have
- [ ] Smooth transitions and animations
- [ ] Professional typography
- [ ] Consistent spacing
- [ ] Optimized images
- [ ] Clean, commented code

### Nice-to-Have
- [ ] Custom icons or illustrations
- [ ] Micro-interactions
- [ ] Scroll animations
- [ ] Video background
- [ ] Dark mode

**Use this rubric to self-evaluate before submitting!** Aim for 90+ points to have a portfolio-worthy project.
