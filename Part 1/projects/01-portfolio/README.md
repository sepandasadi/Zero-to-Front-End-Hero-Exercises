# Project 1: Personal Portfolio Website

**Difficulty**: Beginner
**Estimated Time**: 4-6 hours

## Project Description

Build a complete personal portfolio website to showcase your skills and projects. This is one of the most important projects for any front-end developer as it serves as your online presence and demonstrates your abilities to potential employers or clients.

## Learning Objectives

- Create a multi-section single-page website
- Implement smooth scrolling navigation
- Build responsive layouts with Flexbox and Grid
- Add form validation with JavaScript
- Create interactive UI elements
- Practice mobile-first design

## Requirements

### HTML Structure

- **Hero Section**: Your name, title/tagline, and call-to-action button
- **About Section**: Professional bio and profile photo
- **Skills Section**: List or grid of your technical skills
- **Projects Section**: At least 3 project cards with images, descriptions, and links
- **Contact Section**: Contact form with validation
- **Navigation**: Sticky header with navigation links
- **Footer**: Copyright and social media links

### CSS Styling

- Modern, professional design
- Fully responsive (mobile, tablet, desktop)
- Smooth scrolling between sections
- Hover effects on links, buttons, and project cards
- Grid or Flexbox layout for projects
- Custom color scheme
- Professional typography

### JavaScript Functionality

- Mobile menu toggle (hamburger menu)
- Form validation (name, email, message)
- Smooth scroll to sections when clicking navigation links
- Active navigation link highlighting based on scroll position
- **Optional**: Typing animation in hero section
- **Optional**: Scroll animations (fade in on scroll)

## Starter Files

The `starter/` folder contains:
- Basic HTML structure
- Blank CSS file with comments
- Blank JavaScript file with TODO comments
- Placeholder images

## Getting Started

1. Review the requirements above
2. Look at the starter files in the `starter/` folder
3. Plan your approach (sketch layout, list features)
4. Start with HTML structure
5. Add CSS styling (mobile-first)
6. Implement JavaScript functionality
7. Test thoroughly
8. Compare with solution
