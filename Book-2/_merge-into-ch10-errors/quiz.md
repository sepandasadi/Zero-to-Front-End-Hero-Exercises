# Chapter 28: Error Handling and Debugging - Quiz

Test your knowledge of error handling and debugging concepts!

---

## Questions

### 1. What does `try...catch` do?
- A) Prevents all errors from occurring
- B) Catches errors and prevents app crashes
- C) Makes code run faster
- D) Validates user input

### 2. When does the `finally` block run?
- A) Only if there's an error
- B) Only if there's no error
- C) Always, regardless of errors
- D) Never, it's deprecated

### 3. What's the purpose of throwing custom errors?
- A) Make code harder to read
- B) Enforce business rules and make problems visible
- C) Slow down execution
- D) Replace try/catch

### 4. What does `console.table()` do?
- A) Creates HTML tables
- B) Displays arrays/objects as formatted tables in console
- C) Only works with table elements
- D) Crashes if given invalid data

### 5. What's a breakpoint?
- A) Where your code breaks
- B) A pause in code execution for debugging
- C) An error message
- D) A console.log statement

### 6. What does "Step Into" do in DevTools?
- A) Skip the current function
- B) Enter the function being called
- C) Exit the current function
- D) Continue to next breakpoint

### 7. What's a ReferenceError?
- A) Variable is misspelled or doesn't exist
- B) Wrong data type
- C) Syntax mistake
- D) Network problem

### 8. What's the most common JavaScript error?
- A) SyntaxError
- B) TypeError: Cannot read property of null/undefined
- C) ReferenceError
- D) RangeError

### 9. What does optional chaining (`?.`) do?
- A) Chains multiple functions
- B) Safely accesses nested properties without crashing
- C) Makes properties optional in objects
- D) Converts to strings

### 10. What's the difference between `||` and `??`?
- A) They're the same
- B) `??` only checks for null/undefined, `||` treats 0 and '' as falsy
- C) `||` is newer
- D) `??` is faster

### 11. What's a guard clause?
- A) Security feature
- B) Early return to avoid nested conditions
- C) Error handling
- D) Type checking

### 12. When should you use `console.error()` instead of `console.log()`?
- A) Never, they're the same
- B) For errors - shows in red with stack trace
- C) Only for network errors
- D) Only in production

### 13. What's the call stack?
- A) A type of array
- B) Chain of function calls showing how you got to current point
- C) Error messages
- D) Memory storage

### 14. What does `debugger;` do in code?
- A) Creates a bug
- B) Pauses code execution if DevTools are open
- C) Deletes code
- D) Nothing, it's a comment

### 15. What's defensive programming?
- A) Writing code that protects against errors
- B) Securing against hackers
- C) Making code run faster
- D) Using try/catch everywhere

---

## Answers

1. **B** - Catches errors and prevents app crashes
2. **C** - Always runs, perfect for cleanup
3. **B** - Enforces rules and makes problems visible
4. **B** - Displays data as formatted tables
5. **B** - Pauses execution for inspection
6. **B** - Enters the function to debug it
7. **A** - Variable doesn't exist or is misspelled
8. **B** - The infamous null/undefined property access
9. **B** - Safely accesses nested properties
10. **B** - `??` is more precise for null/undefined checks
11. **B** - Early return pattern for cleaner code
12. **B** - For errors, shows in red
13. **B** - Shows the chain of function calls
14. **B** - Pauses execution like a breakpoint
15. **A** - Anticipating and preventing errors
