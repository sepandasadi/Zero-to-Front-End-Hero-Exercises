## üí° Pro Tips

**API Development Best Practices:**
- Always use try/catch for error handling
- Show loading states while fetching
- Handle errors gracefully with user-friendly messages
- Use async/await instead of .then() chains
- Cache data when appropriate
- Debounce search inputs to reduce API calls
- Check response.ok before parsing JSON
- Use DevTools Network panel to debug requests

**Common Pitfalls to Avoid:**
- ‚ùå Not checking `response.ok` before parsing
- ‚ùå Forgetting to `await response.json()`
- ‚ùå Not handling errors (try/catch)
- ‚ùå Making too many API calls (implement caching)
- ‚ùå Exposing API keys in client-side code
- ‚ùå Not showing loading states
- ‚ùå Poor error messages ("Error" vs "Unable to fetch weather. Please try again.")
