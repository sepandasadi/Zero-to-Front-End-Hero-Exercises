# Magazine Article Layout - Starter Files

## What's Included

- **index.html** - HTML structure with comments and TODO markers
- **css/styles.css** - CSS variables, reset, and starter structure
- **js/script.js** - JavaScript template with TODO comments
- **images/** - Guidelines for images (see images/README.md)

## Getting Started

### 1. Choose Your Article Topic
Pick an interesting topic to write about (800+ words):
- Technology trends
- Design principles
- Personal essay
- How-to guide
- Opinion piece

### 2. Gather Content
- Write or find your article text
- Find high-quality images (see images/README.md)
- Choose fonts (Google Fonts recommended)
- Select a color scheme

### 3. Development Workflow

**Phase 1: Structure (Day 1)**
- [ ] Complete HTML structure
- [ ] Add all article content (800+ words)
- [ ] Add figures and captions
- [ ] Set up grid layout

**Phase 2: Typography (Day 1-2)**
- [ ] Choose and implement font pairing
- [ ] Set up type scale with CSS variables
- [ ] Style drop cap
- [ ] Fine-tune readability (line length, spacing)

**Phase 3: Layout (Day 2)**
- [ ] Implement CSS Grid for main layout
- [ ] Create sidebar
- [ ] Add multi-column sections
- [ ] Style table of contents

**Phase 4: Visual Elements (Day 2-3)**
- [ ] Style figures and captions
- [ ] Create pull quotes
- [ ] Add image grid
- [ ] Style blockquotes

**Phase 5: Interactive (Day 3)**
- [ ] Implement progress bar (JavaScript)
- [ ] Add back-to-top button
- [ ] Create dark mode toggle
- [ ] Test smooth scrolling

**Phase 6: Polish (Day 3)**
- [ ] Add responsive breakpoints
- [ ] Test on mobile/tablet
- [ ] Add transitions and animations
- [ ] Optimize images
- [ ] Validate HTML/CSS

## Tips

### Typography
- Use a type scale generator: https://type-scale.com
- Limit to 2-3 font families maximum
- Body text minimum 18px for readability
- Line length should be 60-75 characters

### Colors
- Start with a primary color and build from there
- Use a color palette generator if needed
- Ensure WCAG AA contrast (4.5:1 minimum)
- Test dark mode carefully

### Layout
- Desktop-first for this project (easier for complex grid)
- Test responsive early and often
- Don't be afraid to hide elements on mobile

### JavaScript
- Keep it simple - focus on CSS skills
- Test in multiple browsers
- Add console.logs for debugging
