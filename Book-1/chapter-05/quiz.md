### Answers:

1. **B** - Exactly one `<h1>` per page (represents the main topic)
2. **B** - `<strong>` has semantic meaning (importance), `<b>` is purely visual
3. **C** - `<aside>` is for tangentially related content like sidebars
4. **B** - Skipping levels breaks document outline and confuses assistive technology
5. **C** - `<dl>` (description list) is perfect for term-definition pairs
6. **B** - `<nav>` contains major navigation link sections
7. **C** - Use divs only when no semantic alternative exists
8. **B** - `<main>` represents the dominant content of the document
9. **B** - `<article>` is independent/reusable, `<section>` groups related content
10. **A, B, and C** - Semantic HTML helps SEO, accessibility, and maintainability (not necessarily load speed)
